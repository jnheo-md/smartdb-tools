SmartDB Tools

Command-line interface and MCP server for the YSR3 Stroke Registry

Access clinical data from the YSR3 multi-center stroke registry through a terminal CLI or AI-assisted IDE integration. Supports schema exploration, patient data queries, outcome analysis, and data export across participating hospitals. Requires an authorized registry account.

Overview

Registry Access

Query patient data, schemas, and clinical variables from the YSR3 database through a structured API. Browse hospitals, tables, and variable definitions across the registry.

Data Export

Export filtered cohorts to XLSX for analysis in R, Python, SAS, or other statistical software. Supports variable selection and filter conditions.

MCP Integration

Integrate with AI-assisted development environments via the Model Context Protocol. Use natural language to query the registry through compatible tools.

Installation

Prerequisites

  • Python 3.10 or later
  • macOS or Linux
  • Authorized YSR3 registry account

Quick Install

Run the installer script to set up SmartDB Tools and its dependencies:

curl -fsSL https://raw.githubusercontent.com/jnheo-md/smartdb-tools/master/install.sh | bash

Post-Install

Authenticate with your registry credentials and verify access:

ysr3 login ysr3 whoami

Usage

Schema Exploration

Browse hospitals, tables, and variables. Search across clinical data points with fuzzy matching.

ysr3 schema hospitals ysr3 schema search YSU "blood pressure" ysr3 schema variable YSU NIHSS_total

Patient Data Queries

Query patient data with automatic table joins. Filter by any variable with support for complex conditions.

ysr3 query data YSU \ --vars "pt_sex,pt_age,NIHSS_total" \ --filters '[{"variable":"Thr_mechanical","operator":"=","value":"1"}]'

Follow-up Outcomes

Retrieve mRS outcomes with automatic death imputation from the cohort table.

ysr3 query followup YSU --period 3m ysr3 query followup YSU --period 12m --limit 20

Clot Composition Data

Access thrombus histology data from the ARIA study, including platelet, fibrin, RBC, and neutrophil composition.

ysr3 clot patients YSU ysr3 clot composition YSU YSU001

Data Export

Export filtered datasets to XLSX for downstream analysis.

ysr3 export data YSU --vars "pt_sex,pt_age,NIHSS_total" ysr3 export followup YSU -p 3m -v "pt_sex,pt_age"

Supported Environments

SmartDB Tools can be used directly in the terminal or integrated with AI-assisted environments via the Model Context Protocol (MCP).

Environment Interface Notes
Terminal CLI Direct command-line access with formatted output
Claude Code MCP Integration with Claude's CLI agent
Claude Desktop MCP Desktop application integration via MCP config
Cursor MCP IDE integration for AI-assisted queries

For MCP configuration details, refer to the MCP setup documentation in the repository.

Data Security

SmartDB Tools does not store or cache patient data locally. All queries are executed through an authenticated API layer that enforces institutional access controls. No database credentials are transmitted to or stored on the client.

  • HTTPS-only API access — all data transmission is encrypted in transit. No direct database connections.
  • Role-based authorization — server-side permission enforcement limits data access to the user's authorized scope and institutional affiliation.
  • Encrypted session storage — JWT tokens are stored with restricted file permissions (0600) and expire automatically after 24 hours.

See SECURITY.md for the full security policy.
